# Competitive Analog Audit (2026)

Goal: identify interaction patterns worth adopting for a unified Junas UX.

## Baseline Analogs Requested

### ChatGPT (OpenAI)
What stands out:
- Project-scoped memory/context for long-running work.
- Unified entry point with tools available inside one conversation surface.
- File + conversation context co-located in project workflows.

Primary sources:
- https://help.openai.com/en/articles/10169521-projects-in-chatgpt
- https://openai.com/chatgpt/overview/

### Perplexity
What stands out:
- Search-native answer flow with citations and source transparency.
- "Spaces" pattern for team context, files, connectors, and scoped retrieval.
- Enterprise emphasis on security controls and data boundaries.

Primary sources:
- https://www.perplexity.ai/help-center/en/articles/10352973-what-is-enterprise-pro
- https://www.perplexity.ai/hub/faq/what-are-spaces
- https://www.perplexity.ai/hub/blog/product

### Harvey
What stands out:
- Unified assistant framing for research, reasoning, and drafting in one interface.
- Workflow agents that reduce prompt burden via structured input steps.
- Legal-specific output controls (citations, step traces, export/sharing).

Primary sources:
- https://www.harvey.ai/platform/assistant
- https://help.harvey.ai/articles/assistant-workflows

## Additional Analogs Recommended (Legal Domain)

### Thomson Reuters CoCounsel
Why relevant:
- Legal-professional trust model with sourced, traceable answers.
- Document analysis + drafting + review under one brand interaction model.

Source:
- https://www.thomsonreuters.com/ai

### LexisNexis Lexis+ AI / Protégé
Why relevant:
- Unified drafting experience across research + drafting + office integration.
- Strong precedent for legal-specific "single flow" interaction.

Source:
- https://www.lexisnexis.com/en-us/solutions/drafting/legal.page

## What Junas Should Borrow
1. Project/workspace container (ChatGPT + Perplexity Spaces).
2. Citation-first output contract with source-side panel (Perplexity + Harvey).
3. Guided workflows for high-risk legal tasks (Harvey + CoCounsel patterns).
4. Single conversation/composer entry point that can invoke all tools.

## What Junas Should Avoid
1. Pure page-per-tool navigation as default.
2. Command catalogs with unsupported actions.
3. URL-based submission for large/sensitive legal text.
4. Duplicate client layers that split behavior by route type.

## Bottom Line
Harvey alone is not enough as a reference. A stronger target combines:
- ChatGPT's project-oriented context model,
- Perplexity's transparent answer/citation UX,
- Harvey/CoCounsel/Lexis legal workflow specialization.

That blend is the right model for a legal AI product that must be both usable and trustworthy.
